<h1 align="center">ST-SillyStyles</h1>

<p align="center">
  Менеджер пресетов стилей для <a href="https://github.com/0xl0cal/sillyimages">sillyimages</a> и его форков.<br>
  Сохраняй стили с превьюшками и переключайся между ними одним кликом.
  Поддерживает NanoBanana и GPT Image 2
</p>

---

## Что это

Вспомогательное расширение для [SillyTavern](https://github.com/SillyTavern/SillyTavern), работающее «над» расширением генерации картинок **[sillyimages](https://github.com/0xl0cal/sillyimages)** (все права на само расширение генерации картинок остаются за автором — это лишь удобная надстройка).

Расширение **перехватывает запросы на генерацию картинок на лету** и подменяет блок `[STYLE: …]` в промпте на стиль из активного пресета. **История чата не трогается** — исходные теги `data-iig-instruction` остаются как есть, модификация происходит только в момент отправки запроса к API.

Совместимо как с оригинальным `sillyimages`, так и с его форками, поддерживающими тот же формат тегов и OpenAI/Gemini-совместимые эндпоинты генерации.

## Фичи

- 🎨 **Библиотека пресетов** со своими превью-картинками
- ✏️ Редактор каждого пресета: **Style** (обязательное), **Начало промпта** и **Конец промпта** (опционально)
- ✅ **Чекбоксы** для множественного выбора — удобно массово удалять или экспортировать
- 📦 **Импорт / Экспорт JSON**: экспорт одного, нескольких (по галочкам) или всех сразу; импорт добавляет к существующим
- 🚫 **Не трогает историю чата** — подмена стиля происходит на уровне HTTP-запроса к API генерации
- 🔌 Работает **параллельно** с sillyimages и его форками, ничего в них патчить не надо

## Установка

В SillyTavern: **Extensions → Install extension** → вставить URL:

```
https://github.com/niemandswasser/ST-SillyStyles
```

Либо вручную: склонировать репозиторий в `SillyTavern/public/scripts/extensions/third-party/ST-SillyStyles`.

После установки перезагрузи страницу SillyTavern. В настройках расширений появится блок **🖌️ ST-SillyStyles**.

## Как пользоваться

1. Открой **🖌️ ST-SillyStyles → 🎨 Библиотека стилей**.
2. Введи название нового пресета → **Добавить**.
3. В редакторе заполни:
   - **Style** *(обязательное)* — попадёт в блок `[STYLE: …]` запроса на генерацию.
   - **Начало промпта** *(опционально)* — допишется в самое начало.
   - **Конец промпта** *(опционально)* — допишется в самый конец.
   - **Превью** — загрузи картинку для удобной визуальной идентификации.
4. Кликни по карточке в сетке, чтобы активировать пресет (зелёная рамка + галочка).
5. Следующий запрос на генерацию картинки через sillyimages автоматически использует твой стиль.

Повторный клик по активной карточке снимает активацию — тогда расширение не вмешивается в запросы.

### Множественные операции

Отметь пресеты галочками в левом верхнем углу карточек. В панели над сеткой:
- **Выбрать все / Снять выбор**
- **Экспорт (N)** — экспортировать только отмеченные
- **Удалить (N)** — удалить отмеченные

Есть также отдельные кнопки на карточке (при наведении): экспорт одиночного пресета и удаление.

### Импорт / Экспорт

В секции **💾 Импорт / Экспорт**:
- **Экспорт всех** — выгрузить всю библиотеку одним JSON-файлом.
- **Импорт JSON** — загрузить файл; пресеты **добавляются** к существующим (дубликатов по id не создаётся, каждому присваивается новый id).

Поддерживаемые формы JSON при импорте: `{presets: [...]}`, `{stylePresets: [...]}` или просто массив `[...]`.

## Как это работает (технически)

Расширение монки-патчит глобальный `window.fetch`. При каждом вызове:
1. Проверяет, что URL ведёт на эндпоинт генерации картинок:
   - OpenAI-совместимый: `…/v1/images/generations`
   - Gemini: `…/v1(beta)?/models/<model>:generateContent`
2. Если да и активен пресет — парсит тело запроса, находит регуляркой блок `[STYLE: …]` / `[Style: …]` в поле `prompt` (OpenAI) или в `contents[].parts[].text` (Gemini).
3. Заменяет на `[STYLE: <твой стиль из пресета>]`. Если блока в промпте не было — добавляет в начало.
4. Опционально добавляет **Начало промпта** в начало и **Конец промпта** в конец.
5. Отправляет модифицированный запрос.

Исходное сообщение в `context.chat[n].mes` **не модифицируется** — в истории чата остаётся оригинальный тег `data-iig-instruction` с первоначальным стилем от LLM.

## Формат экспорта

```json
{
  "app": "ST-SillyStyles",
  "version": 2,
  "exportedAt": "2026-04-22T12:34:56.789Z",
  "presets": [
    {
      "id": "sstyle_...",
      "name": "Манга нуар",
      "style": "masterpiece, 8k, high contrast black and white manga...",
      "promptStart": "",
      "promptEnd": "Adult 18+ fictional characters",
      "previewImage": "<base64 PNG>",
      "createdAt": 1761130000000
    }
  ]
}
```

`previewImage` — base64 без префикса `data:`, ресайз до 512px по длинной стороне.
